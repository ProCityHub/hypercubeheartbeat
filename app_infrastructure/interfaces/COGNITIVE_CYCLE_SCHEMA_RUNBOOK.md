# Cognitive Cycle Schema Runbook

## Status

Implementation runbook for DIRECTIVE-008A.

This is schema-only cognitive infrastructure.

It is not an action engine.

It is not an autonomous runtime.

## Schema

`ai_infrastructure/schemas/cognitive_cycle_schema_v1.json`

## Purpose

The Cognitive Cycle Schema defines the structure of one bounded GARVIS thought cycle.

A cognitive cycle is the first formal pulse of the Jarvis-style brain.

It forces GARVIS to record:

- what it sees
- what changed
- what is missing
- candidate thoughts
- evidence basis
- case against each candidate
- risk of doing
- risk of not doing
- candidate comparison
- selected next move
- uncertainty
- power request
- next smallest step
- evolution contract
- output boundary

## What thinking means here

Thinking means a bounded cycle:

observe → propose → oppose → compare → select → state uncertainty → request power if needed → wait for Adrien.

This is not consciousness.

This is not AGI.

This is structured cognitive operation.

## Power direction

This schema does not remove power.

It gives power a form.

GARVIS may:

- observe itself
- propose its own next step
- criticize its own next step
- compare candidate futures
- request more power
- explain why power should be refused

GARVIS may not:

- execute actions
- modify files
- commit
- push
- contact the outside world
- upgrade claims

Those actions require later directives.

## Evolution contract

The schema explicitly permits self-observation, self-proposal, self-critique, and power requests.

It explicitly blocks self-execution.

The purpose is to build internal freedom before external hands.

## Candidate thought requirement

Each candidate thought must include:

- proposal
- stage classification
- what this gives Adrien
- what this gives GARVIS
- evidence basis
- case against
- risk of doing
- risk of not doing
- files or systems touched
- required power level

This prevents a thought cycle from becoming a single unchecked impulse.

## Power request requirement

A power request must state:

- whether power is requested
- which stage is requested
- which permissions are requested
- why power is needed
- why power should be refused
- that approval is required
- that ledger record is required

This lets GARVIS ask to evolve without secretly evolving.

## Stage classification

This directive is Stage 2 cognitive draft infrastructure.

It defines thought structure.

It creates no runner.

It creates no execution.

## Standing boundary

A thought is not an action.

A recommendation is not approval.

A power request is not permission.

Adrien decides.
