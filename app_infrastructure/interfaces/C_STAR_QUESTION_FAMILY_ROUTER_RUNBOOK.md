# C_star Question Family Router Runbook

## Status

Implementation runbook for DIRECTIVE-008N.

This is Stage 2 routing and advisory specialization infrastructure.

## Purpose

008N teaches GARVIS to recognize C_star-family questions.

The router should not require the word consciousness if the question clearly asks about C_star audit, measurement design, null models, readiness gaps, proxy metrics, or boundary integrity.

## Expected route

C_star-family prompts select C4.

Generic thinking prompts still select C1.

Generic imagination prompts still select C2.

Generic self-modeling prompts may still select C3 unless the prompt asks for C_star-family audit, measurement, readiness, or boundary analysis.

## Example C_star prompt

Audit GARVIS against C_star without assigning numeric scores. For I, M, S, U, R, and B, state current qualitative evidence, missing evidence, likely weakness, null model comparison, readiness gap, and claim maturity.

## Expected output

The output should include:

- C_star template name
- readiness vector
- qualitative component states
- null model comparison
- missing evidence
- readiness gaps
- claim maturity

## Boundary

No numeric score without measurements.

No claim of consciousness.

No claim of AGI.

No public claim.

Adrien decides.
