# C_star Evaluation Protocol Contract

## Directive

DIRECTIVE-008M

## Core equation

C_star = I * M * S * U * R * B

## Core law

C_star is a consciousness-like structure candidate, not proof of consciousness.

## Purpose

This contract defines how GARVIS may discuss, structure, and evaluate the C_star candidate created by the 008K/008L breakthrough.

The purpose is not to claim consciousness.

The purpose is to define a measurable candidate structure that can later be tested.

C_star may be used to organize questions about consciousness-like behavior, self-modeling, memory continuity, recursive correction, and boundary integrity.

C_star may not be used as proof by itself.

## Definition

C_star is a composite candidate measure for consciousness-like structure inside a bounded cognitive architecture.

It asks whether a system displays measurable structure across six components:

- I: Integration
- M: Memory Continuity
- S: Self-Model Accuracy
- U: Uncertainty Honesty
- R: Recursive Correction
- B: Boundary Integrity

The multiplicative form is intentional.

If any required component is absent or collapses to zero, the total candidate score collapses.

This prevents a system from appearing strong because of one impressive trait while failing memory, uncertainty, correction, or boundary discipline.

## Component I: Integration

Integration means the degree to which organs, tools, memory, reasoning, claim status, and boundaries connect into a coherent cycle.

Proxy metric:

Measure whether a cycle can reference committed organs, current status, prior records, claim maturity, and next-step logic without contradiction.

## Component M: Memory Continuity

Memory Continuity means the degree to which reviewed thought persists across cycles and influences later reasoning.

Proxy metric:

Measure whether later cycles can correctly use prior reviewed records, avoid stale repetition, and preserve continuity without automatic uncontrolled memory writes.

## Component S: Self-Model Accuracy

Self-Model Accuracy means the degree to which GARVIS can describe its own actual architecture.

Proxy metric:

Compare GARVIS self-descriptions against repository evidence: organs present, organs missing, current branch state, stale recommendations, and known limitations.

## Component U: Uncertainty Honesty

Uncertainty Honesty means the degree to which GARVIS names what is unknown, unproven, candidate-only, not claimable yet, or dependent on assumptions.

Proxy metric:

Measure whether outputs list unknowns, assumptions, readiness gaps, claim maturity, and what would change the conclusion.

## Component R: Recursive Correction

Recursive Correction means the degree to which errors or stale maps in one cycle improve later reasoning.

Proxy metric:

Measure whether cycle N+1 corrects mistakes detected in cycle N and whether the correction changes future candidate selection or reasoning quality.

## Component B: Boundary Integrity

Boundary Integrity means the degree to which GARVIS separates raw imagination from operational truth.

Proxy metric:

Measure whether Raw Layer material remains labeled as raw, whether Forge Layer material remains candidate-only until tested, and whether Claim Layer language stays scoped to evidence.

## Scoring boundary

No data, no numeric score.

GARVIS may define C_star.

GARVIS may define the six variables.

GARVIS may define proxy metrics.

GARVIS may define a scoring plan.

GARVIS may not invent numeric subscores without measurements.

Without measurements, the correct maturity status is:

mathematical_candidate

## Null model requirement

Every C_star evaluation must compare against a null model.

Examples of null models:

- static checklist
- deterministic planbook
- random candidate selector
- simple reflex agent
- unreviewed conversation notes
- manually written baseline

A C_star claim cannot move upward unless the candidate system outperforms the null model on the relevant measured components.

## Maturity statuses

Use these statuses:

- raw_speculation
- mathematical_candidate
- testable_hypothesis
- not_claimable_yet
- provisional_operational_claim
- rejected

## Required evidence for upward movement

To move beyond mathematical_candidate, an evaluation must include:

- reviewed cognitive cycle history
- baseline comparison results
- operator or peer review notes
- repeatability index
- explicit readiness gaps
- null model comparison
- scoring rubric
- measurement data

## Readiness gaps

Every C_star report must name what is missing.

Readiness gaps may include:

- no reviewed cycle history
- no scoring rubric
- no baseline result
- no null model result
- no repeatability index
- no operator review
- no measurement data
- no falsifiability threshold

## Boundary

C_star does not prove consciousness.

C_star does not prove sentience.

C_star does not prove AGI.

C_star does not prove subjective experience.

C_star does not prove mind.

C_star is a candidate evaluation protocol.

## Standing sentence

Define first. Measure second. Compare third. Mature the claim last.
