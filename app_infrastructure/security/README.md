# App Security

## Purpose

This folder will hold future app security rules.

Current status: skeleton only.

Security defaults:

- no secrets in repository
- no raw sensor data in repository
- no hidden network calls
- no unapproved external actions
- no autonomous sending
- no automatic contact with people
- no silent claim changes
- no bypassing GitHub memory
- no bypassing guard checks

Future app work must define threat model, permission model, and approval model before adding capability.
