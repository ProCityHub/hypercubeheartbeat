# Consciousness Operational Definition

## A. Position statement

This project does **not** claim consciousness.

This project does **not** claim completed AGI.

This project defines measurable properties that may be tested as possible
signatures of consciousness-like organization in a deterministic cognitive
architecture.

Absence of these properties is publishable.

Failure is not damage to the project. Failure is evidence.

The goal is to define what would count as a measurable signal, what would count
as a control, and what would count as a written failure condition before any
future claim is made.

## B. Operational properties

### LL-10 — Coherence maintenance

**Operational question:**  
Can the system maintain coherent state organization under contradiction,
interruption, or noisy input without collapsing into arbitrary output?

**Metric:**  
Measure the stability of diagnostic reports across repeated perturbations:
limiting constraint, limit origin, settle/bounce action, and warning structure.
A coherent system should preserve interpretable diagnostic structure under
controlled perturbation.

**Control:**  
Compare against a shuffled-state control and a random-warning control that
receive the same inputs but do not preserve Observer/Actor/Bridge structure.

**Failure condition:**  
LL-10 fails if coherence scores, limiting constraints, or warnings are no more
stable than shuffled-state or random-warning controls under matched
perturbation.

### LL-11 — Identity persistence

**Operational question:**  
Can the system preserve a recognizable self-state across time without merely
copying the previous output?

**Metric:**  
Measure whether habit signatures, cube-corner transitions, and diagnostic
state summaries persist across repeated related inputs while still allowing
state change when inputs change.

**Control:**  
Compare against a stateless diagnostic run and a direct-copy baseline.

**Failure condition:**  
LL-11 fails if identity-like persistence is indistinguishable from stateless
recomputation or direct copying of prior output.

### LL-12 — Self-model accuracy

**Operational question:**  
Can the system accurately report what it can do, what it cannot do, and why a
claim is unsupported?

**Metric:**  
Measure agreement between the system's diagnostic self-report and external
test outcomes: PASS/FAIL for software self-tests, SUPPORTED/NOT_SUPPORTED for
pre-registered empirical tests, and explicit uncertainty when evidence is
insufficient.

**Control:**  
Compare against a report-only baseline that always emits confidence language
and a conservative baseline that always emits uncertainty.

**Failure condition:**  
LL-12 fails if the system's self-report does not reliably match external test
outcomes or if it overclaims unsupported results.

### LL-13 — Integration

**Operational question:**  
Can the system integrate observation, action, bridge state, memory, and
language into one coherent diagnostic report without losing claim discipline?

**Metric:**  
Measure whether Observer, Actor, Bridge, cube corner, heartbeat phase,
coherence score, habit signatures, limitation class, warnings, and evidence
are all present and mutually consistent across tasks.

**Control:**  
Compare against ablated reports missing one or more of Observer, Actor, Bridge,
memory, or evidence fields.

**Failure condition:**  
LL-13 fails if integrated reports do not outperform ablated reports in
predicting task outcome, limitation class, or claim-validity category.

## C. Linguistic layer

The linguistic layer includes symbolic and relational meaning, including
wahkohtowin.

In this project, wahkohtowin is treated as a relational and interpretive
framework for understanding connectedness, responsibility, kinship, and
meaning-making.

This layer is studied as a phenomenon **of** consciousness-like organization,
not as a tested proof **of** consciousness.

The linguistic layer is currently registered under LL-07 as an interpretive
framework and possible semantic coordinate system.

It is not a validated result.

It must not be used as evidence that consciousness has been proven.

## D. What voids any consciousness claim

Any future consciousness-related claim is void if any of the following occur:

1. Metrics are changed after seeing results.
2. Analyses are run without registration and then described as confirmatory.
3. Failed, negative, null, or inconvenient results are removed.
4. Software self-tests are described as empirical support.
5. PASS/FAIL self-test language is confused with SUPPORTED/NOT_SUPPORTED
   empirical-test language.
6. The system claims consciousness, completed AGI, or physics validation
   without pre-registered evidence.
7. Interpretive language, symbolic language, or wahkohtowin framing is treated
   as a substitute for measurement.
