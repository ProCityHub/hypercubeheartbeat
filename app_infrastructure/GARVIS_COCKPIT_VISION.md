# GARVIS Cockpit Vision

## Status

Vision document.

Inspirational only.

Not a contract.

Not an implementation plan.

Not an app.

Not an action authorization.

## Purpose

This document captures the cockpit vision for a future GARVIS app.

The cockpit should give Adrien more scientific power, more control, and more visibility.

It should not give GARVIS hidden autonomy.

## Design phrase

Simple surface.

Hard boundary.

Beautiful control.

No hidden hands.

## Future cockpit surfaces

A future GARVIS cockpit may include:

- Home status
- current GARVIS mode
- Stage 1 ledger view
- approval state view
- proposal inbox
- case-against display
- action queue
- audit trail
- tests and guard status
- infrastructure map
- claim ledger view
- evidence map
- experiment dashboard
- advisory AI review inbox
- self-design suggestions
- app/tool roadmap

## Scientific power direction

The cockpit should help Adrien:

- see the system clearly
- inspect memory
- compare proposals
- run reproducible checks
- track claims
- audit evidence
- prepare research
- avoid accidental claim upgrades
- govern actions
- stop actions
- review advisor disagreement
- build tools one organ at a time

## Boundary

The cockpit may display.

The cockpit may propose after a future directive.

The cockpit may queue approvals after a future directive.

The cockpit may execute actions only after a future directive.

This vision does not authorize implementation.

## Standing sentence

The cockpit should make Adrien more powerful.

It should not make GARVIS less governable.
